Glossar der Begriffe und Abkürzungen

Projekt: Linked Open Exhibition — NFDI4Culture / Hochschule Hannover (BIM-126-02)
Zuletzt aktualisiert: April 2026

A

AAT — Art & Architecture Thesaurus
Ein strukturiertes Vokabular des Getty Research Institute zur Beschreibung von Kunst, Architektur und materieller Kultur. In diesem Projekt als kontrolliertes Vokabular für Linked-Art-Typklassifikationen verwendet.

API — Application Programming Interface (Programmierschnittstelle)
Eine definierte Schnittstelle, über die Softwareanwendungen miteinander kommunizieren. In diesem Projekt werden folgende APIs genutzt: DNB-SRU-API, Wikidata Query Service, Lobid-GND-API und die Wikibase MediaWiki Action API.

Attribution (KI)
Alle Notebooks und Dokumentationen in diesem Projekt nennen GitHub Copilot (Claude Sonnet 4.6) als verwendetes KI-Werkzeug, entsprechend der KI-Attributionsrichtlinie des Projekts.

B

BIM-126-02
Der Kurscode für Data Science (SoSe 2026) an der Hochschule Hannover, geleitet von Simon Worthington und Prof. Ina Blümel. Der Kurs ist eine achtteilige Einführung in Linked Open Data für GLAM unter Nutzung von Wikimedia-Foundation-Plattformen.

Bot-Konto
Ein dediziertes Wikibase/MediaWiki-Benutzerkonto für automatisierte Datenuploads über wikibaseintegrator oder mwclient. Zugangsdaten werden in einer .env-Datei gespeichert (niemals in die Versionskontrolle eingecheckt).

C

CC0 — Creative Commons Zero
Die Gemeinfreiheits-Widmungslizenz, die auf DNB-Metadaten angewendet wird. Über die DNB-SRU-API abgerufene Datensätze werden unter CC0 1.0 Universal veröffentlicht und können ohne Namensnennung frei weitergenutzt werden.

CC BY-SA 4.0 — Creative Commons Namensnennung-Weitergabe unter gleichen Bedingungen 4.0 International
Die Lizenz für Projektinhalte (Folien, Dokumentation). Bei Weiternutzung sind Namensnennung und Weitergabe unter gleichen Bedingungen erforderlich.

CIDOC-CRM — Comité International pour la Documentation — Conceptual Reference Model
Ein ereignisbasiertes Ontologiemodell für die Dokumentation des kulturellen Erbes, gepflegt von ICOM. Linked Art ist ein Anwendungsprofil von CIDOC-CRM 7.1.3. Kernklassen umfassen E7_Activity, E22_Human-Made_Object, E53_Place und E55_Type.

CQL — Contextual Query Language
Eine Standardabfragesprache für das SRU-Protokoll. DNB-SRU-Abfragen verwenden CQL-Syntax, z. B. KOR="Sprengel Museum" and SWW="Ausstellungskatalog".

CRMdig
Eine Erweiterung von CIDOC-CRM zur Beschreibung digitaler Provenienz. Stellt Klassen wie D1_Digital_Object und D29_Annotation_Object bereit, die in der Linked-Art-Ontologie referenziert werden.

CRMsci
Eine wissenschaftliche Beobachtungserweiterung von CIDOC-CRM, die Klassen wie S19_Encounter_Event zur Beschreibung von Objektfunden bereitstellt.

CSV — Comma-Separated Values (Kommagetrennte Werte)
Ein einfaches tabellarisches Textformat für Zwischenausgaben wie sprengel_exhibitions.csv sowie zur Kodierung des WB4R-Datenmodells (wikibase_generic_model.csv).

D

DNB — Deutsche Nationalbibliothek
Die Nationalbibliothek Deutschlands, die einen zentralen bibliografischen Katalog pflegt, der auch Ausstellungskataloge enthält. Die DNB-SRU-API und die Library-Lab-Dienste sind primäre Datenquellen in diesem Projekt.

DNB Library Lab
Ein Programm der DNB, das experimentellen API-Zugang, Jupyter-Notebook-Tutorials und Datendienste für Entwicklerinnen und Entwickler sowie Forschende bereitstellt. Siehe: https://www.dnb.de/librarylab

DNB-SRU-API
Die DNB-Implementierung des SRU-Protokolls (Search/Retrieve via URL). Endpunkt: https://services.dnb.de/sru/dnb. Bietet kostenlosen, registrierungsfreien Zugriff auf bibliografische Datensätze im MARC21-XML-Format.

Dot / Graphviz DOT
Eine Klartextsprache zur Graphbeschreibung, verwendet in der Datei data.dot zur Darstellung von Datenbeziehungen. Wird durch Graphviz oder als {dot}-Codeblock in Quarto gerendert.

E

ENV-Datei (.env)
Eine Datei zur Speicherung umgebungsspezifischer Konfigurationswerte wie API-Zugangsdaten. Die .env-Datei ist per .gitignore ausgeschlossen; die eingecheckte .env.example dokumentiert die erforderlichen Schlüssel ohne reale Werte preiszugeben. Zur Laufzeit über python-dotenv geladen.

F

FOAF — Friend of a Friend
Ein RDF-Vokabular zur Beschreibung von Personen und ihren Beziehungen. Im WB4R-Datenmodell als äquivalente Klasse für Akteure referenziert (foaf:Agent).

G

GLAM — Galleries, Libraries, Archives and Museums (Galerien, Bibliotheken, Archive und Museen)
Ein Sammelbegriff für Kulturerbeeinrichtungen, die kulturelle Objekte sammeln, bewahren und zugänglich machen. Das Projekt adressiert GLAM-Anwendungsfälle, insbesondere Kunstmuseumsausstellungen.

GND — Gemeinsame Normdatei
Die deutschsprachige Normdatei, gepflegt von der DNB und Netzwerkpartnern. Stellt stabile URIs für Personen, Organisationen, Orte und Schlagwörter bereit. Zugänglich über die Lobid-GND-API.

GitHub Pages
Ein statischer Website-Hosting-Dienst von GitHub, über den die Quarto-Website-Ausgabe aus dem Verzeichnis docs/ veröffentlicht wird. URL: https://nfdi4culture.github.io/linked-open-exhibition/

I

IDN — Identifikationsnummer
Der eindeutige Datensatzidentifikator im DNB-Katalog. Kodiert im MARC21-Feld 001. Zur Erstellung von Cover-Bild-URLs und zur Verknüpfung von Datensätzen systemübergreifend genutzt.

IRI — Internationalized Resource Identifier (Internationalisierter Ressourcenbezeichner)
Eine Verallgemeinerung des URI/URL-Standards mit Unicode-Unterstützung. In RDF und Ontologien zur eindeutigen Identifikation von Entitäten und Eigenschaften verwendet, z. B. https://www.cidoc-crm.org/cidoc-crm/E22_Human-Made_Object.

ISBN — International Standard Book Number
Ein numerischer Bezeichner für veröffentlichte Bücher. Kodiert im MARC21-Feld 020. Zum Abruf von Coverbildern über die DNB-Cover-Bild-API genutzt (https://portal.dnb.de/opac/mvb/cover?isbn={ISBN}).

J

JSON-LD — JavaScript Object Notation for Linked Data
Eine Methode zur Kodierung von Linked Data in JSON-Syntax. Die Linked-Art-API serialisiert ihre Datensätze als JSON-LD 1.1. Wikibase unterstützt ebenfalls JSON-LD-Export.

JupyterLite
Eine browserbasierte Jupyter-Umgebung, die Python (via Pyodide/WebAssembly) ohne Server ausführt. Das DNB Library Lab betreibt eine JupyterLite-Instanz unter https://deutsche-nationalbibliothek.github.io/jupyterlite/lab/. Die früheren Notebooks des Projekts in catalogues/dnb-jupyter/ wurden für JupyterLite konzipiert.

L

LDS — Linked Data Service (DNB)
Der Dienst der DNB zum Herunterladen von Metadaten in Bulk als RDF oder andere Linked-Data-Formate. Siehe: https://www.dnb.de/EN/Professionell/Metadatendienste/Datenbezug/LDS/lds_node.html

LIDO — Lightweight Information Describing Objects
Ein XML-Schema für die Bereitstellung, Ernte und den Austausch von Museumsobjektdaten. Beispieldateien liegen im Verzeichnis LIDO/. Als Referenzformat für die Beschreibung von Kulturerbe-Objekten verwendet.

Linked Art
Eine internationale Initiative und ein Datenmodell, das ein gemeinschaftlich gepflegtes Anwendungsprofil von CIDOC-CRM für die Beschreibung von Kunst und Kulturerbe bereitstellt. Serialisiert als JSON-LD 1.1. Version 1.0. Siehe: https://linked.art/

Lobid-GND
Eine API des Hochschulbibliothekszentrums (hbz), die GND-Normdaten als JSON-LD bereitstellt. Unterstützt schnelle Autovervollständigung und URI-basierte Abfragen. Dokumentation in LOD-API/Lobid-GND.md.

LOD — Linked Open Data
Daten, die im Web mit offenen Standards (RDF, HTTP, URIs) veröffentlicht und mit anderen Datensätzen verknüpft werden. Das Projekt demonstriert LOD für Kunstausstellungen durch Verknüpfung von DNB-, Wikidata-, GND- und Wikibase-Daten.

LOD-API
Ein Verzeichnis im Projektrepository mit Dokumentation und Beispielen für die Arbeit mit Linked-Open-Data-APIs, insbesondere Lobid-GND.

M

MARC21 — MAchine-Readable Cataloging (Version 21)
Ein bibliografischer Metadatenstandard in Bibliothekssystemen. DNB-SRU-API-Antworten werden als MARC21-XML geliefert. Im Projekt verwendete Schlüsselfelder: 001 (IDN), 245 (Titel), 264/260 (Erscheinungsjahr), 020 (ISBN), 689 (Schlagwörter) und 856 (URL).

MediaWiki
Die Wiki-Software, die Wikipedia und Wikibase zugrunde liegt. Das Projekt nutzt mwclient für den Upload von Bildern in die MediaWiki-Instanz des Projekts.

Mermaid
Eine JavaScript-basierte Diagrammsyntax zur Erstellung von Flussdiagrammen und Sequenzdiagrammen aus Klartext. Verwendet in exhibition.mmd, flow.mmd und durch Quarto gerendert.

mwclient
Eine Python-Bibliothek für die Interaktion mit MediaWiki-APIs. In diesem Projekt zum Upload von Coverbildern in die MediaWiki-Instanz des Projekts verwendet.

N

NFDI — Nationale Forschungsdateninfrastruktur
Eine deutsche Bundesinitiative zum Aufbau einer Forschungsdateninfrastruktur für verschiedene wissenschaftliche Disziplinen.

NFDI4Culture
Das NFDI-Konsortium für die Erforschung des materiellen und immateriellen Kulturerbes. Das Projekt wird unter dem Dach von NFDI4Culture durchgeführt.

O

OWL — Web Ontology Language
Ein W3C-Standard zur Definition von Ontologien im Semantic Web. Das WB4R-Datenmodell verwendet OWL-Klassendefinitionen (owl:Class) zur Beschreibung von Entitätstypen.

P

python-dotenv
Eine Python-Bibliothek, die Schlüssel-Wert-Paare aus .env-Dateien in Umgebungsvariablen einliest. Zum Laden von Wikibase-Zugangsdaten zur Laufzeit ohne Hardcodierung verwendet.

Q

QID — Q-Item-Identifikator
Ein eindeutiger Bezeichner für ein Element in Wikidata oder Wikibase, mit dem Präfix Q (z. B. Q510144 für das Sprengel Museum). Das Q-Präfix wurde von den ursprünglichen Wikidata-Entwicklern arbiträr gewählt, um Element-Bezeichner von Eigenschafts-Bezeichnern (die P verwenden) und Lexem-Bezeichnern (die L verwenden) zu unterscheiden. Der Buchstabe selbst trägt keine semantische Bedeutung — er war einfach ein Buchstabe, der im technischen Kontext noch nicht gebräuchlich war. QIDs sind stabil und sprachunabhängig, was sie zu zuverlässigen Bezeichnern für die systemübergreifende Verknüpfung von Daten unabhängig von der Label-Sprache macht. In SPARQL-Abfragen und Datenmodell-Mappings in diesem Projekt referenziert.

Quarto
Ein quelloffenes wissenschaftliches und technisches Publikationssystem (Version ≥ 1.3), das zur Erstellung der Projektwebsite aus Markdown-Quellen (.qmd) und Jupyter-Notebooks verwendet wird. Ausgabe wird für GitHub Pages in docs/ veröffentlicht.

R

RDF — Resource Description Framework
Ein W3C-Standard zur Darstellung von Informationen als Subjekt-Prädikat-Objekt-Tripel. Die Grundlage von Linked Data und SPARQL.

RevealJS
Ein JavaScript-Präsentationsframework. Die Projektdatei index.qmd wurde ursprünglich als RevealJS-Folien formatiert, bevor sie in eine Quarto-Website umgewandelt wurde.

S

SPARQL — SPARQL Protocol and RDF Query Language
Eine Abfragesprache für RDF-Daten. In diesem Projekt zur Abfrage von Wikidata und dem Wikibase-Endpunkt des Projekts verwendet. Beispielabfragen visualisieren Ausstellungen als Zeitachsen, Graphen und Karten.

SPARQLWrapper
Eine Python-Bibliothek zum Senden von SPARQL-Abfragen an RDF-Endpunkte und zur Verarbeitung der Ergebnisse. In requirements.txt als Projektabhängigkeit aufgeführt.

Sprengel Museum Hannover
Ein Kunstmuseum in Hannover, das sich auf moderne und zeitgenössische Kunst spezialisiert. Die primäre Anwendungsfalleinrichtung des Projekts, das den aus DNB-Datensätzen gewonnenen Ausstellungsdatensatz bereitstellt.

SRU — Search/Retrieve via URL
Ein OASIS/Library-of-Congress-Standardprotokoll für bibliografische Suche und Datensatzabruf über HTTP. Abfragen werden in CQL geschrieben, Ergebnisse als XML zurückgeliefert. Die DNB betreibt einen SRU-Endpunkt unter https://services.dnb.de/sru/dnb, der kostenlosen, registrierungsfreien Zugriff auf den gesamten DNB-Katalog im MARC21-XML-Format bietet. Paginierung über den Parameter startRecord (bis zu 100 Datensätze pro Anfrage). Die von der DNB verwendete Protokollversion ist SRU 1.1.

SWW — Schlagwort
Ein CQL-Indexfeld für DNB-SRU-Abfragen zur Suche nach Schlagwörtern. SWW="Ausstellungskatalog" filtert nach Ausstellungskatalog-Datensätzen.

U

URI — Uniform Resource Identifier (Einheitlicher Ressourcenbezeichner)
Eine Zeichenkette zur eindeutigen Identifikation einer Ressource. In Linked Data werden URIs als Bezeichner für Entitäten, Eigenschaften und Konzepte verwendet, um die Verknüpfung zwischen Datensätzen zu ermöglichen.

W

WB4R — Wikibase for Research (Generisches Modell)
Ein Referenz-Datenmodell, entwickelt von NFDI4Culture (Rossenova, Sohmen, Duchesne), das einen Satz generischer Klassen und Eigenschaften für eine forschungsorientierte Wikibase-Instanz definiert. Repository: https://gitlab.com/nfdi4culture/wikibase4research/model/. Die CSV-Datei WB4R/wikibase_generic_model.csv dient in diesem Projekt als Referenz für Eigenschafts- und Klassen-Alignments.

Wikibase
Eine quelloffene Softwareplattform (Teil der Wikimedia-Suite) zur Erstellung strukturierter, verlinkter Wissensbasen. In diesem Projekt wird die gehostete Instanz unter https://wikibase.wbworkshop.tibwiki.io/ zur Speicherung von Ausstellungsmetadaten verwendet.

wikibaseintegrator
Eine Python-Bibliothek zum Lesen und Schreiben von Daten in Wikibase-Instanzen über die MediaWiki-API. Für den programmatischen Upload von Ausstellungsdatensätzen verwendet.

Wikidata
Eine freie, kollaborative, mehrsprachige Wissensbasis der Wikimedia Foundation. Studierende haben in frühen Kurssitzungen Ausstellungseinträge manuell in Wikidata angelegt. Öffentlicher Wikidata-SPARQL-Endpunkt: https://query.wikidata.org/

Wikimedia Commons
Ein Repository frei lizenzierter Mediendateien (Bilder, Audio, Video) der Wikimedia Foundation. Als Teil des Kursplattform-Kontexts referenziert.

Wikiversity
Ein Wikimedia-Foundation-Projekt für offene Bildungsinhalte. Kursmaterialien für BIM-126-02 sind hier gehostet: https://en.wikiversity.org/wiki/BIM-126-02-Data-Science-Linked-Open-Exhibition

X

XML — Extensible Markup Language (Erweiterbare Auszeichnungssprache)
Ein strukturiertes Textformat für den Datenaustausch. MARC21-XML ist das von der DNB-SRU-API zurückgegebene Format. LIDO-Dateien in diesem Projekt sind ebenfalls XML-basiert.

Dieses Glossar umfasst Begriffe und Abkürzungen aus den Projektdateien, Datenmodellen, Notebooks und der Dokumentation. Für Ontologieklassendefinitionen siehe Linkedart/linked-art-terms.md und Linkedart/linked-art-ontology.md. Für WB4R-Eigenschaftsdetails siehe WB4R/wikibase_generic_model.csv.
