Exhibition Catalogue OCR Pipeline

Extracting structured artwork data from scanned PDF catalogues

Technical documentation for the three-stage pipeline that OCRs a scanned museum catalogue and produces a structured CSV ready for Wikibase import.

Overview

This document describes the pipeline for digitising structured data from scanned art museum exhibition catalogues. The primary source is scans/baldessari-sprengel.pdf, a scanned exhibition catalogue from the Sprengel Museum Hannover.

The pipeline has three stages:

Stage Script Input Output
1 – OCR scripts/ocr_catalogue.py scans/*.pdf output/ocr/pages/*.txt, output/ocr/*_ocr_full.txt
2 – Extract scripts/extract_artworks.py OCR full text output/baldessari-sprengel-artworks.csv
3 – Validate scripts/validate_csv.py CSV Console quality report

An interactive JupyterLab notebook (catalogues/catalogue-ocr.ipynb) combines all stages with data exploration visualisations.

Prerequisites

Python packages

Install via pip install -r requirements.txt:

pdf2image
pytesseract
Pillow
pymupdf
pandas

Tesseract OCR engine

Tesseract must be installed separately on your operating system.

Windows: Download and run the installer from the UB Mannheim repository. During installation, tick the language packs German (deu) and English (eng).

After installation, if Tesseract is not on your PATH, set the path explicitly in Python:

import pytesseract
pytesseract.pytesseract.tesseract_cmd = r"C:\Program Files\Tesseract-OCR\tesseract.exe"

Linux/macOS:

sudo apt install tesseract-ocr tesseract-ocr-deu tesseract-ocr-eng   # Debian/Ubuntu
brew install tesseract                                                   # macOS

Poppler (required by pdf2image)

Windows: Download from poppler for Windows and add the bin/ directory to your PATH.

Linux: sudo apt install poppler-utils
macOS: brew install poppler

Usage

Command-line scripts

Stage 1 – OCR:

python scripts/ocr_catalogue.py scans/baldessari-sprengel.pdf \
  --output output/ocr \
  --lang deu+eng \
  --dpi 300

Options:

Flag Default Description
--output output/ocr Directory for OCR artefacts
--lang deu+eng Tesseract language string
--dpi 300 Rendering DPI (≥300 recommended)
--pages all Page range, e.g. 1-30 or 5,10,15

Stage 2 – Extract:

python scripts/extract_artworks.py output/ocr/baldessari-sprengel_ocr_full.txt \
  --output output/baldessari-sprengel-artworks.csv \
  --exhibition "Baldessari. Sprengel Museum Hannover" \
  --location "Sprengel Museum Hannover" \
  --start 2000-09-17 \
  --end 2001-01-14

Stage 3 – Validate:

python scripts/validate_csv.py output/baldessari-sprengel-artworks.csv

Jupyter notebook

Open catalogues/catalogue-ocr.ipynb in JupyterLab and run cells sequentially. The notebook includes interactive data quality visualisations.

Parsing Strategy

Exhibition catalogue entries typically follow a repeating block pattern:

JOHN BALDESSARI
Big Red Arm with Thumb, 2000
Chromogenic print
76 × 61 cm
Sprengel Museum Hannover
Kat.-Nr. 5

The extract_artworks.py parser uses the following heuristics:

  1. Artist name — a line in ALL-CAPS (≥4 characters) starts a new entry block.
  2. Dimensions — any line matching N×N or N x N patterns.
  3. Medium — lines containing material keywords (print, photograph, oil, Druck, etc.).
  4. Collection — lines containing institution keywords (Museum, Sammlung, Courtesy, etc.).
  5. Catalogue number — lines matching Kat.-Nr., Cat., No., or [N] patterns.
  6. Year — a standalone four-digit number, or a year suffix on the title line.
  7. Title — the first unmatched line after the artist name.

Values that cannot be assigned with confidence are flagged with [?] for manual review.

Manual Review

After Stage 2, open output/baldessari-sprengel-artworks.csv in LibreOffice Calc, Excel, or similar:

  1. Remove [?] suffixes from corrected values.
  2. Fix obvious OCR artefacts (confused characters: 0/O, 1/l, rn/m, etc.).
  3. Fill in artist_wikidata_qid for known artists (e.g. Q318159 for John Baldessari).
  4. Add artist_gnd_id values where available from the GND authority file.
  5. Verify catalogue_no matches the printed catalogue numbering.
  6. Re-run Stage 3 to confirm the quality score is satisfactory.

Future Work: DTD and XML Import

Once the CSV quality is confirmed, the following will be created:

  • output/artwork-catalogue.dtd — document type definition describing the XML structure, aligned with the Linked Open Exhibition data models.
  • output/baldessari-sprengel-import.xml — one <item> element per artwork, structured for batch import into the NFDI4Culture Wikibase instance.

The DTD design and XML generation scripts will be documented in a separate notebook (catalogues/wikibase-xml-import.ipynb) when data quality is sufficient.
